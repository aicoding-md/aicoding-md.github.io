# 采摘园网当前实现说明

## 项目定位

当前项目已经从原先的 AI Skill 展示站，重构为面向 `caizhaiyuan.com` 的全静态采摘园信息站。

站点核心目标：

- 首页展示中国各省份精选的 1 个代表城市
- 城市卡片展示该城市的采摘特色聚合信息
- 城市详情页展示该城市下的多个采摘园卡片
- 每个采摘园卡片包含：
  - 采摘园名称
  - 地址
  - 电话
  - 亮点介绍
  - 信息来源
- 全站使用静态 HTML，方便搜索引擎收录

## 当前页面结构

### 1. 首页

入口文件：

- `index.html`

首页内容结构：

- 品牌区：`采摘园网`
- Hero 主视觉
- 城市卡片列表
- 每个城市卡片展示：
  - 省份
  - 城市
  - 城市采摘主题
  - 城市摘要
  - 主打水果标签

首页定位是“城市级采摘特色聚合页”，不是单个采摘园详情页。

### 2. 城市详情页

输出目录：

- `cities/*.html`

每个城市详情页包含：

- 城市级介绍
- 采摘园卡片清单
- 适合采摘什么
- 游玩亮点
- 出行建议
- 侧边信息栏

采摘园不再使用表格，而是使用卡片式结构，移动端更友好。

## 当前数据组织方式

主数据文件：

- `scripts/orchards-source.mjs`

这个文件目前承担了 3 类数据职责：

### 1. 站点基础信息

- `siteMeta`
- 包括站点标题、描述、首页主图

### 2. 图片映射

- `imageMap`
- 不同水果或农场主题对应不同图片

### 3. 城市与采摘园数据

- `destinations`
- 每个城市由 `makeDestination(...)` 生成

每个城市包含：

- `slug`
- `province`
- `city`
- `title`
- `crops`
- `season`
- `summary`
- `description`
- `pick`
- `highlights`
- `address`
- `contact`
- `tips`
- `searchKeywords`
- `orchards`

其中 `orchards` 是城市下的采摘园列表。

## 采摘园数据来源逻辑

项目当前有两套采摘园来源机制：

### 1. 已核实采摘园

使用：

- `verifiedOrchardsBySlug`

当某个城市已经补充了真实园区信息时，会优先读取这部分数据。

每个采摘园对象结构为：

```js
{
  name,
  address,
  phone,
  crop,
  highlight,
  source,
  sourceUrl
}
```

说明：

- `source` 用于展示文字来源
- `sourceUrl` 用于跳转到公开信息页面
- 页面上电话会自动转成 `tel:` 链接

### 2. 默认占位采摘园

使用：

- `makeOrchards(...)`

如果某个城市还没有补入真实园区数据，就会自动生成默认的 6 个采摘园卡片。

这类卡片当前特点：

- 数量固定为 6
- 电话显示为 `电话待核实`
- 主要用于先把页面结构铺齐

## 当前数量规则

目前已经统一了全站采摘园卡片数量规则：

- 每个城市至少 6 个采摘园
- 数量必须是双数
- 当前全站统一为 6 个

对应实现已经写在数据层，生成时会自动带出。

## 页面生成方式

生成脚本：

- `scripts/generate-site.mjs`

当前脚本负责生成：

- `index.html`
- `cities/*.html`
- `sitemap.xml`
- `robots.txt`

生成逻辑要点：

- 读取 `destinations`
- 遍历生成所有城市详情页
- 写出首页和 SEO 文件
- 自动清理旧的 `cities` 目录并重建

运行方式：

```bash
node scripts/generate-site.mjs
```

## 推荐维护方式

后续维护建议遵循下面的顺序：

1. 先在 `scripts/orchards-source.mjs` 中补充或修改城市采摘园数据
2. 优先补 `verifiedOrchardsBySlug`
3. 每条真实电话都尽量补上来源和来源链接
4. 修改完成后重新执行生成脚本
5. 检查对应 `cities/*.html` 是否已更新

## 不建议直接修改的内容

不建议直接手改这些生成产物：

- `index.html`
- `cities/*.html`
- `sitemap.xml`
- `robots.txt`

因为这些文件会在重新生成时被覆盖。
